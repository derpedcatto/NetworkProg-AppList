﻿SMTP - Simple Mail Transfer Protocol - протокол отправки эл. почты
Как это работает
Задача: из Приложения отправить сообщение на почту Клиента

Приложение                   Email-service (Gmail)                     Клиент
           Посылка сообщения                                          (client@i.ua)
Подключение <----------------->
Авторизация  -- логин/пароль --> app@gmail.com
            <----- OK ----------
Cообщение ----To: client@i.ua--> Прием сообщения
            <----- OK ----------            -- To: client@i.ua -----> Новое письмо от app@gmail.com
                                               From: app@gmail.com      (код или ссылка)
Проверка кода<-------------заход в приложение и ввод кода----------------




Реализация:
1. нужно создать почтовый ящик app@gmail.com на почтовом сервисе, поддерживающем SMTP
   (в доступном пакете услуг)
2. настроить возможность доступа по SMTP: далее на примере Gmail
  - создаем новую учетную запись (не рекомендуется использовать одну почту для 
     нескольких проектов. Это усложнит их продажу/передачу заказчику)
  - в настройках профиля указываем двухэтапную аутентификацию (Безопасность - Вход в аккаунт Google)
     после этого появляется вкладка "Пароли приложений"
  - создаем пароль для приложения: Тип приложения (Почта), Устройство (Компьютер Windows)
     копируем пароль и сохраняем, дальше он будет недоступен
     этот пароль будет использоваться для аутентификации (вместо пароля от учетки)
3. сохраняем данные для почты в отдельном файле конфигурации (emailconfig.json)
  - создаем в проекте файл emailconfig.json
  - исключаем этот файл из репозитория (.gitigone - дописываем имя файла(ов) )
  = для целостности проекта можно создать в репозитории emailconfig.json с
      шаблонными строками "change me" (зайти на github, добавить файл, вместо
      логинов/паролей заменить шаблонами)
  - делаем файл доступным в папке с исполнимым кодом: в обозревателе решений
      запрашиваем свойства emailconfig.json (при фокусировке файла появляются снизу)
      Меняем параметр "Copy to Output Directory" -- "Copy if newer"
4. Для возможности подтверждения пароля нужно хранить информацию об отправленном
     коде, причем это хранение должно быть долгосрочным и "выдерживать" перезагрузку
     программы. Добавляем в программу БД (проект - добавить новый эл-т - БД на службах)
